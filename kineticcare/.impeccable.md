# KineticCare — Design Context

*This file is the design brief for the KineticCare site under `kineticcare/`. It is NOT for the parent `SuryaSg.github.io` portfolio, which is a separate brand.*

## Design Context

### Users

**Primary users** are Bengaluru adults presenting with musculoskeletal complaints — typically:

- **Tech workers** (25–45) with chronic cervical, thoracic, or lumbar issues from 8–10+ hours of desk work per day. They're analytical, they Google their own symptoms, and they've usually seen 2–3 physios before us. They want *measurement*, not reassurance.
- **Athletes and active adults** recovering from ACL, rotator cuff, meniscus, Achilles-tendon injuries. They want return-to-sport metrics, not rest prescriptions.
- **Women seeking pelvic floor, prenatal, postnatal, or menopause care** — an underserved population in Indian physio. They want specialist training, discretion, and female clinicians on request.
- **Older adults (55+)** with neurological rehab needs — post-stroke, Parkinson's, balance. Often arriving with a caregiver. They want patience and a pace that matches theirs.
- **Corporate HR leads** evaluating team-wide ergonomics programs. They want aggregate MSK metrics, ROI justification, and a single point of contact.

**Context of use**: mostly phones (commute, late-night pain-googling), some desktop (HR research, booking). Rarely in the clinic itself — the site is the *pre-clinic* experience.

**Job-to-be-done**: *"Tell me whether this clinic can actually fix my specific thing, with specific numbers, and let me book without a sales funnel."*

### Brand Personality

Three words, in tension with each other — that tension is the brand:

- **Clinical** — precise, evidence-based, quantitative. Numbers with units. Tabular numerals. Actual vertebra names (C5–C7, L4–L5). Not "we care", but "median recovery 3.2× faster".
- **Confident** — no hedging copy, no disclaimers scattered everywhere, no "may help with some conditions". Specific claims. The tone of a senior clinician giving you a direct answer.
- **Warm** — terracotta, cream, hand-drawn anatomical linework. Syne's geometric-but-friendly display weight. No hospital blue, no latex-glove sterility. The clinic is in Indiranagar, not a corporate medical campus.

**Emotional goal**: patients should feel *trusted with information*, not *sold to*. The site should remove the "consulting is just sales" anxiety that most clinic sites produce.

### Aesthetic Direction

**Established visual language** (already in the codebase — do not rebuild from scratch):

- **Palette**: Terracotta (`#C96B4A` light / `#E0896A` dark) as the single chromatic accent. Turquoise (`#2BB5A4`) as the secondary clinical signal. Warm cream (`#FAF8F3`) and warm off-black (`#0E0B08`) for surfaces. 5-tier ink scale from `--ink` through `--muted2`.
- **Typography**: Syne (display 400/600/700/800), DM Sans (body 300/400/500), JetBrains Mono (instrument readouts, metadata, eyebrows). Smart quotes, tight display letter-spacing, tabular-nums on all numerics.
- **Shadow system**: Multi-layer brand-tinted shadows (not neutral gray). Terracotta tinted far + neutral close = chromatic depth.
- **Signature pattern — "clinical instrument"**: monospace labels, thin border rules, live pulsing dots, vertebra labels, tabular numerals, minus signs. The hero's "LIVE · 42 in program · 12,427 recoveries · 3.2× faster · 94% adherence" bar is the archetype.
- **Signature pattern — "wireframe anatomy"**: the `drawSpine()` canvas on the landing page and the `generate.html` 3D viewer. Stroke-based linework, not filled shapes. Should be extended and referenced when new visual elements are needed.
- **Motion**: slow, deliberate. 200–400ms ease-out-quart / cubic-bezier(.16,1,.3,1). Respects `prefers-reduced-motion`. No bounce, no elastic. Pulse rings breathe at ~2s cycles.

**Theme default**: light mode. Dark mode fully supported and tested — user choice is persisted to `localStorage['kc-theme']`. Dark is honored for users browsing in late-night pain.

**Anti-references (what this is NOT)**:

- Not generic AI UI: **no** cyan-on-dark hero, **no** purple-to-blue gradients, **no** glassmorphism everywhere, **no** identical rounded-card grids, **no** gradient text.
- Not Web3/crypto: **no** neon glow, **no** dark-with-gradient-overlay, **no** mirrored logos, **no** futuristic sans.
- Not generic SaaS: **no** hero → 3-feature-grid → testimonials → pricing-carousel template, **no** Stripe purple, **no** bento layouts, **no** dashboard-in-hero.
- Not medical/corporate: **no** hospital blue, **no** stock stethoscope photography, **no** Arial/Open Sans, **no** stiff "Our Mission" sections.

**Anti-pattern enforcement**:

- Never `border-left: Npx solid` (the universal medical/admin tell).
- Never gradient text via `background-clip: text`.
- Never "gray text on colored backgrounds" — use a shade of the background hue instead.
- Never pill buttons with icon-left-big-number-label-small metric cards. The clinical bar is the metric treatment.

### Design Principles

Five principles, in order of precedence (earlier wins when they conflict):

1. **Measurement over reassurance**. Every claim has a number. Every progress indicator has units. Vague copy ("may help") gets replaced with specific ranges ("4–12 weeks with evidence-based protocol"). If we can't state it in numbers, we don't state it.

2. **Instrument, not marketing**. The site reads like a clinical readout — tabular numerals, live indicators, monospace labels, vertebra names. The user should feel like they're reading a dashboard, not a brochure. When in doubt, render as monospace before reaching for a display font.

3. **Anatomical linework as signature**. Wireframe bones, ellipse ribcages, stroke-based organs. This is the one thing competing clinics don't have. Extend it — every new visual element should ask "can this reference the spine/skeleton aesthetic?" before reaching for generic iconography.

4. **Warm clinical, not cold clinical**. Every terracotta use must be earned — if removing the accent leaves the page readable, keep it removed. But warmth is non-negotiable. Cream backgrounds, 5-tier ink scale (never pure black or pure white), brand-tinted shadows.

5. **Accessible by default (WCAG AA)**. 4.5:1 body contrast, 3:1 UI contrast, focus-visible rings, keyboard orbit on interactive canvas, `prefers-reduced-motion` respected in every animation block, semantic headings, skip links on every page, tabular numerals stable when counters animate.

### Required stack constraints

- Plain HTML/CSS/JS. No framework. No build step. No npm.
- Each page is self-contained with inline `<style>` and `<script>`.
- Design tokens as CSS custom properties in `:root` + `[data-theme="dark"]`.
- Three.js r128 loaded from CDN (with fallback chain: cdnjs → jsdelivr → unpkg).
- Google Fonts: Syne, DM Sans, JetBrains Mono.
- GitHub Pages hosting. Must degrade gracefully for slow connections and disabled WebGL.

### When to deviate

These principles hold unless the user explicitly says "this is experimental" or "push past the brand". Then anything goes — but document which principle was intentionally broken so we can decide whether to roll it back into the system or keep as a one-off.
